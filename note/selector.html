1.基本選擇器

    1-1類型選擇器

        a.元素選擇器(Type selectors)：tagName 選元素(p div...等等)
            div{color:red;}

        b.class選擇器(Class selectors)：className 選class
            .box{color:red;}

        c.ID選擇器(ID selectors)：#idName 選id
            #box{color:red;}

        d.通用選擇器(Universal selectors): * 全選
            d-1: *{} 全部都會有{}裡的效果
            *{padding:0;}
            d-2: .box+*{} .box後面那個兄弟會有{}的效果
            .box{box}*2+.box1{box1}+.box{box} css: .box+*{color:red}

*只要是.box後面的那個兄弟都會有效果，但.box1後面就不會有喔!

        e.屬性選擇器Attribute selectors： [] 選屬性

            e-1:[屬性]
                a[href]{color:red;} a標籤具有href屬性的文字就會變紅色
                href="#123

            e-2:[屬性=某個值] 屬性正好等同"某個值"
                a[href="#"]{color:red;} a標籤具有href="#"屬性的文字就會變紅色
                href="#"

            e-3:[屬性~=某個值] 如果屬性中，有用空格分隔，只要其中有一個符合"某個值"即可。
                a[class~="blue"]{color:yellow;} a標籤的class只要兩個或兩個以上，其中一個為"yellow"即可達到效果
                class="red blue" href="#"

            e-4:[屬性|=某個值] 屬性中，恰好等同"某個值"，或以"某個值"開頭，用-連接，也都符合。
                a[class|="purple"]{color:purple;} a標籤的class開頭purple就可以吃到
                class="purple-color" href="#"

            e-5:[屬性^=某個值] 屬性中，以"某個值"為前綴
                a[class^="gray"]{color:gray;} a標籤的class開頭purple就可以吃到
                class="gray-color" href="#"
                然後就發現e-4跟e-5作用差不多。(歡迎實驗報告ˊˇˋ)

            e-6:[屬性$=某個值] 屬性中，以"某個值"為後綴
            a[href=".tw"]{color:darkred;} a標籤的href最後是".tw"就可以吃到
                href="http://yahoo.com.tw"

            e-7:[屬性*=某個值] 屬性中，"某個值"有出現過
                a[href*=4]{color:orange;} a標籤具有href屬性的文字就會變橘色
                href="#1234

            e-8:[屬性*=某個值 i] 屬性中，"某個值"不分大小寫的比較
                a[href="TW" i]{color:darkblue;} a標籤的href最後是".tw"就會變成深藍色
                href="tw"

            e-9:[屬性*=某個值 s] 屬性中，"某個值"區分大小寫的比較
                a[href="ORG" s]{color:darkblue;} 跟e-8同樣的狀況下，用i就不會有效果，而是吃到a[href]的紅色
                href="org"

        f.狀態選擇器State selectors: : 狀態偽類選擇器 這部分留到偽類一起

    1-3偽類選擇器Pseudo蘇斗 classes:在特殊狀態下被選取的話，會作為關鍵字被加到選擇器裡面。
        a.元素顯示狀態偽類Element display state pseudo-classes：這些偽類可以根據元素的顯示狀態來選擇元素。
            *這部分目前還沒碰到，先放著。
            a-1: :fullscreen 處在全螢幕模式下
            a-2: :modal 看不懂。
            a-3: :picture-in-picture 跟影音處理有關，可以做成子母視窗(未來再研究)
        b.輸入偽類Input pseudo-classes：這些偽類與表單元素相關，可以根據 HTML 屬性和交互前後字段的狀態來選擇元素。
            b-1: :autofill 當表單選項是自動填的狀態會有的樣式


當自動輸入名字時，會變藍框*input內設定name=""
input:autofill {border: 3px solid blue}


            b-2: :enabled 啟用的部分


當input是enable時，會變黃色背景*input預設
input:enabled{background-color: yellow;}


            b-3: :disabled 禁用的部分


當input是disable時，會變紅色背景*input內設定disabled
input:disabled{background-color: red;}


            b-4: :read-only 不能更改的部分


當input是read-only，會變綠色背景*input內設定readonly
input:read-only{background-color: green;}


            b-5: :read-write 可編輯任何元素的部分


當input是read-write，會變橘色背景*input預設就是read-write
input:read-only{background-color: orange;}


            b-6: :placeholder-shown placeholder顯示的部分


當input是有placeholder時，會變藍色背景*input內設定placehoder
input:placeholder-shown{background-color: blue;}


            b-7: :default 表單被checked跟提交按鈕的部分


當input被checked或是submit時，會變紫色背景*input內設定type="submit"
input:default{background-color: purple;}


            b-8: :checked 按鈕被checked的時候

   
當input被checked時，會變成粉紅色邊框
input:checked {outline: 2px solid deeppink;}

 
            b-9: :indeterminate 不確定狀態的元素

   
當input還沒被碰過時，會變成深紅色背景 *都不用做就是未確定的狀態。
input type="radio" id="radio1" name="radioButton"

input type="radio" id="radio2" name="radioButton"

input:indeterminate+label {background: darkred;}
*因為選項(radio / checkbox)會有一堆效果吃不到，所以會用+之類的選擇器選去其他的對象

            
            b-10: :blank 空白，或包含一個空字符串或其他空輸入。*目前都沒有瀏覽器支援喔

            b-11: :valid 內容為有效時
        

   
當input有用的時後，會變成深橘色背景
input:valid {background-color: darkorange};

 
            b-12: :invalid 內容為無效時

   
當input輸入的內容無效時，會變成深黃色背景input type="number" value="5" min="18"
input:invalid {background-color: darkyellow};

 
            b-13: :in-range 搭配具範圍限制的元素，在範圍內時 如：number

   
當input輸入的內容在範圍內時，會變成深綠色背景input type="number" value="5" min="18"
input:in-range {background-color: darkgreen};

 
            b-14: :out-of-range 搭配具範圍限制的元素,不在範圍內時

   
當input輸入的內容不在範圍內時，會變成深藍色背景input type="number" value="5" min="18"
input:out-of-range {background-color: darkblue};


            b-15: :required 在input添加required的屬性，即可在CSS設定

   
當input具有required的屬性時，會有紅色虛線效果的框input data-name="effect_required" type="text" required *input內設定type="required"
input:required {border:5px dashed red};


            b-16: :optional 在input未添加任何屬性的，即可在CSS設定

   
當input不具任何屬性時，會有橘色虛線效果的框input data-name="effect_required" type="text" required *input內設定type="required"
input:optional {border:5px dashed orange};

 
            b-17: :user-invalid 目前只有火狐有用，如果填入不合法回應時。

    1-4偽元素選擇器Pseudo elements: CSS偽元素添加到選擇器的關鍵字，可讓設置所選元素的特定部分的樣式
            a. ::after 你的好朋友，在所選的元素增加一個最後的子元素
            b. ::before 你的好朋友，在所選的元素增加一個第一的子元素
            c. ::backdrop 影片全螢幕播放時，本來黑色的背景，就可以用這個改顏色。
            d. ::cue 元素內WebVTT提示，總之不是目前要了解的部分。
            e. ::first-letter 設定第一個字母的樣式
            f. ::first-line 設定第一行的樣式
            g. ::file-selector-button 設定檔案選擇的按鈕樣式(針對input的type是file)
            h. ::marker 用在li，可以設定開頭的樣式
            i. ::part 這個好像很強，等學到shadow DOM時，應該就會知道了。
            j. ::placeholder 你的好朋友，可以預設不感擾的預設字在input上(但好像都在input行內寫了)
            k. ::selection 喜歡，將反白的部分設定樣式
                CSS設定為::selection {color: gold;background-color: red;}
